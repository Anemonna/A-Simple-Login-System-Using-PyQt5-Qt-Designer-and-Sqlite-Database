# A-Simple-Login-System-Using-PyQt5-Qt-Designer-and-Sqlite-Database
这是我使用 Python (PyQt5) 、Qt Designer 和 SQLite 搭建的一个带有用户权限控制的桌面端管理系统。在这个项目中，我采用了 Qt Designer 来实现界面 UI 与核心业务逻辑的有效分离，并使用本地轻量级数据库来持久化存储数据。相关全部文件包括程序以及示例程序已经上传。

---

## 1. 登录与注册模块 (`LoginWindow`)
这是系统的入口，我采用了无边框（Frameless）和透明背景的现代感设计，并通过重写鼠标事件保留了窗口的拖动功能。

### 🔑 登录模块：身份验证与窗口交互

在登录模块的设计中，我首先加载了使用 Qt Designer 绘制的登录界面，并移除了标准边框，配合透明背景营造出扁平化的现代视觉感。当用户点击登录按钮时，程序会立即建立与 `MyData.db` 的本地 **SQLite 数据库连接**，并创建一个游标对象。我编写的逻辑会提取用户输入的账号和密码，通过 SQL 语句进行参数化查询。


如果查询结果为空，系统将 切换至错误提示页，告知用户“账号或密码错误”或“不能为空”。一旦验证通过，程序会执行一系列关键动作：首先将当前的用户名和密码存储在类属性中以供全局调用，接着实例化主窗口并显示，最后销毁登录界面。此外，为了解决无边框窗口无法移动的问题，我重写了 `mousePressEvent` 和 `mouseMoveEvent` 方法，确保用户依然可以随意拖动窗口位置。


### 📝 注册模块：严密的逻辑校验体系

当用户切换到注册页面并提交信息时，程序并不会直接写入数据库，而是会经历一套自上而下的**条件过滤机制**。首先是**完整性检查**，系统会判断账号、密码以及确认密码是否为空，任何一项缺失都会触发“不能为空”的警告。紧接着是**一致性校验**，程序会比对两次输入的密码字符串，只有在完全相同的前提下才会进入下一步，否则会提示“两次密码不一致”，有效防止了用户因误打而导致无法登录的情况。



最核心的一步是**唯一性验证**：程序会再次访问数据库中的 `users` 表，检索该用户名是否已被注册。如果账号已存在，系统会通过提示页拦截此次请求，防止数据覆盖。只有当所有校验全部通过后，程序才会生成一条新的 SQL 插入指令。在插入数据时，我确保在发生突发写入错误时程序不会直接崩溃。注册成功后，界面会自动引导用户返回登录页，完成从新用户到系统成员的转换。

## 2. 系统主界面 (`MainWindow`)
登录后的核心操作台，通过左侧导航栏布局，并实现了基于角色的访问控制 (RBAC)。

### 🏠 首页模块：个性化欢迎与状态显示

作为用户登录后的首个落脚点，**首页**承担了系统导航与状态确认的核心职责。我通过在界面顶部设计动态标签，实现了对当前登录身份的实时渲染——系统会自动从 `LoginWindow` 类属性中读取已验证的用户名并进行展示，增强了系统的交互感。界面中心则配置了清晰的欢迎语引导，作为整个管理系统的视觉基准点。此外，该页面也作为无边框主窗体的承载容器，用户可以在此通过我重写的鼠标事件逻辑，实现对整个 1077x767 分辨率大窗口的平滑拖动与位置调整。

### 🌐 网站推荐：外链集成与便捷导航

为了提升系统的实用性，我专门开发了一个**网站推荐**功能页面。该模块本质上是一个集成化的快捷导航中心，通过调用标准库，我将 Bilibili、GitHub、CSDN 和 Apple 等高频访问的平台入口直接封装在界面按钮中。用户无需手动打开浏览器并输入网址，系统便会自动调用底层系统的默认浏览器并精准跳转至目标 URL。这种设计将桌面客户端的逻辑与 Web 资源无缝连接，极大缩短了用户获取外部信息的路径。

### 🔐 我的信息：全流程密码安全自维护

**我的信息**模块是系统安全体系的重要组成部分，它允许用户在无需管理员介入的情况下完成**密码自助修改**。在逻辑实现上，我设计了一套极为严格的校验闭环：
1. **身份二次验证**：系统会首先连接 `MyData.db`，验证输入的“旧密码”是否与数据库记录完全吻合。
2. **逻辑冲突拦截**：程序会自动比对新旧密码，若用户尝试修改为相同密码，系统会拦截并提示“新密码不能与旧密码相同！”，防止无效操作。
3. **输入一致性检查**：通过双重输入框确认新密码，确保用户不会因误触而设置错误的密码。
一旦所有校验通过，系统会执行 `UPDATE` SQL 指令实时更新数据库。为了确保新安全策略立即生效，我还加入了一个自动化触发逻辑——修改成功后系统将自动调用 `logout` 函数注销当前会话，并重新唤起登录窗口，要求用户使用新凭证重新进入系统。

## 3. 数据查看与管理（管理员专属）
仅有 `admin` 账户可进入此模块，进行系统级数据管理。
### 📊 数据查看与管理：多维度的后台运维中心

在系统的**数据查看与管理**模块中，我为管理员提供了一套深度集成数据库的交互方案。该模块不仅是数据的展示窗口，更是具备完整逻辑校验的运维后台，主要分为“注册用户管理”与“学生信息展示”两大核心支柱。

#### 1. 注册用户管理：实时的 CRUD 交互体系
在注册用户管理页面，我实现了**全自动化的数据同步机制**。系统会从 `users` 表中提取用户名、密码及联系方式，并以可交互表格的形式呈现。
* **内联实时编辑**：通过监测 `itemChanged` 信号，我实现了“所见即所得”的编辑体验。当管理员双击并修改单元格内容时，程序会捕捉当前行的主键信息，自动生成并执行 SQL `UPDATE` 语句，确保本地数据库与界面显示瞬间同步。
* **数据生命周期管理**：界面下方集成了完整的数据录入表单。在新增用户时，系统会自动计算逻辑 ID 以维护数据库的自增一致性；在删除操作中，系统通过精准的行匹配机制，调用 `DELETE` 指令安全地移除目标记录，并实时重刷数据列表，保证了管理操作的闭环。

#### 2. 学生信息展示：高级渲染与智能检索

针对 `Students` 表的大规模数据展示，我侧重于**用户体验（UX）的深度优化**。
* **视觉增强渲染**：为了缓解长时间审阅数据的疲劳，我通过代码逻辑为表格注入了“斑马纹”背景效果，并对不同性质的数据列（如姓名、邮箱、电话）应用了冷暖色调交替的字体渲染，极大地提升了信息的可读性。
* **多模式检索与高亮**：我设计了一个功能强大的搜索过滤器。管理员可以自由切换“全局搜索”或“指定列搜索（如按城市、按学历）”。系统在执行模糊查询后，不仅会过滤数据，还会利用 `setBackground` 方法将所有匹配到的结果行以黄色高亮显示，确保核心信息在复杂数据中跃然纸上。
* **动态排序机制**：通过开启 `setSortingEnabled` 并结合表头点击事件，我实现了一键切换升序（▲）与降序（▼）的功能，方便管理员快速定位极端值（如最大年龄或最新注册日期）。

## 4. 数据库设计 (`MyData.db`)

* **`users` 表**：构建账号认证体系 (`id`, `username`, `password`, `nation`, `telephone`)。
* **`Students` 表**：存储业务数据实体 (`id`, `name`, `age`, `gender`, `city`, `email`, `education`, `birthday`, `phone`)。

---

*项目开发说明：本系统遵循界面与逻辑分离原则，使用 PyQt5 作为 UI 框架，SQLite 作为底层数据库。*
